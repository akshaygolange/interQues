# Controlled vs Uncontrolled Inputs in React

## Core takeaway

- Controlled inputs are preferred when validation, conditional rendering, or form management is needed, while uncontrolled inputs can be useful for simple forms and file inputs.
- In React, controlled inputs store their value in state and are ideal for validation, conditional UI, and complex form logic, while uncontrolled inputs rely on the DOM and are handy for simple forms, file inputs, or when you only need the value on submit.
- Use state (controlled input) to show live value.
- Use refs only in event handlers (submit, button click) for uncontrolled inputs.

---

## Example component

```jsx
import React, { useRef, useState } from "react";

const ControlledVsUncontrolled = () => {
  const [text, setText] = useState("");
  const handleChange = (e) => {
    setText(e.target.value);
  };

  const inputRef = useRef(null);
  const handleClick = () => {
    alert(inputRef.current?.value);
  };

  return (
    <>
      <div>
        <h2>Controlled Input</h2>

        <h2>{text}</h2>
        <input type="text" value={text} onChange={handleChange} />
      </div>

      <div>
        <h2>Uncontrolled Input</h2>
        <input type="text" ref={inputRef} />

        <button onClick={handleClick}>Submit</button>
      </div>
    </>
  );
};

export default ControlledVsUncontrolled;
```

- The first input is **controlled** (`value={text}` + `onChange` → React state is the source of truth).
- The second input is **uncontrolled** (DOM manages the value; we read it via `inputRef.current.value` on button click).

---

## Ref gotcha in JSX

If you write:

```jsx
<h4>{inputRef ? inputRef.current.value : null}</h4>
```

inside the JSX for the uncontrolled input, it can break because:

- On the initial render, `inputRef.current` is `null`, so `inputRef.current.value` throws.
- Even if you guard with `inputRef.current && inputRef.current.value`, React will not automatically re-render when `ref.current.value` changes, since refs don’t cause re-renders.

So:

- Use **state** when you want the UI to update on every keystroke.
- Use **refs** to read values at specific moments (submit, button click) in uncontrolled inputs.
